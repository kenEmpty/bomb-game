# 📋 PROJECT STATUS — ボムサバイバル

爆弾で地形を破壊しながら最後の1人（チーム）を目指す、2〜4人対戦のターン制サバイバルゲーム。
HTML / CSS / JavaScript のみ・外部サーバー不要・単一フォルダで動作（スマホ最優先 / PC対応）。

- 公開URL: https://kenempty.github.io/bomb-game/
- リポジトリ: https://github.com/kenEmpty/bomb-game
- 配信: GitHub Pages ＋ Service Worker による完全オフライン対応
- - バランス調整用の定数: `js/config.js`（移動歩数・爆弾射程・障害物数・マップサイズ・CPU思考時間・アニメ速度・セル縦横比・チーム定義 など）

最終更新: 2026-06-16
現在のアセットバージョン: `?v=9` / SW キャッシュ `bombgame-v9`

## ✅ 実装済み機能

### コアシステム
- [x] マップサイズ選択（小 8×7 / 中 10×9 / 大 12×10）
- [x] 2〜4人対戦・四隅への初期配置

## 🎨 UI改善案

> 詳細レビュー済み。優先度の高い順。

1. **勝利画面に「もう一度（同設定）」ボタン**（周回性が大きく向上・低コスト）
2. **設定画面の整理**：上部に「かんたん開始」、詳細はアコーディオン折りたたみ、デバッグ項目は隠す、カテゴリ見出し（対戦/ルール/サウンド）
3. **オンボーディング**：遊び方ボタン＋初回手番の吹き出しヒント
4. **カラー役割の整理**：UIアクセント色を個人色・チーム色から独立。チーム＝塗り／個人＝番号 に役割分担
5. **射程/移動ハイライトの二重符号化**：色だけでなくアイコン・パターンで区別
6. **自分キャラの常時強調**（リング/矢印）と**破壊済みマスの視認性**（へこみ/影）
7. バッジ・muted文字の最小サイズ／コントラスト見直し
8. アプリアイコンをタイトルロゴに採用しブランド統一
9. メニューにサウンド/BGMトグル・遊び方を集約、設定の localStorage 保存

## 🤖 CPU改善案

1. **爆弾思考の高度化**：現状は「倒せる→逃げ道を削る→安全」の優先順。投擲も移動と同様に「この爆弾で相手を追い込む/将来の撃破を作る」評価へ
2. **読みの深化（Very Hard）**：自分の移動だけでなく、敵の最善応手まで含めた2手読み（ミニマックス的）。経路列挙は枝刈りで軽量化
3. **チーム連携の強化**：味方と挟み込む・敵の退路を分担して塞ぐなど、協調的な布陣
4. **終盤の精度**：盤面が狭くなった局面での自滅回避・詰め筋の優先
5. **撃破リスクと攻めのバランス調整**：現状は生存最優先。状況に応じて積極性を可変に（劣勢時は攻め寄り 等）
6. **CPUの個性/思考時間のばらつき**で対戦感を演出
7. `CPU.WEIGHTS` の調整UI（デバッグ用）で挙動チューニングを容易に
